# Xentral — Information Architecture (the CEO law)

Most ERP/CRM systems are structured by **technical modules** (CRM, Inventory,
Purchasing, Accounting, HR). That is logical for developers — not for founders.
A CEO does not think in modules. A CEO thinks in the **business lifecycle**:

    Lead → Customer → Quote → Order → Delivery → Invoice → Payment → Support

So Xentral is structured by lifecycle, not by modules. This is a permanent law.

## Permanent main navigation (lifecycle order)

- **Mission Control** — Dashboard · Ask Xentral AI · Timeline · Calendar · Work Queue
- **Revenue** — Leads · Contacts · Companies · Deals · Pipelines · Quotes · Forecasting
- **Operations** — Orders · Products · Inventory · Warehouses · Procurement · Suppliers · Projects
- **Finance** — Invoices · Payments · Receivables · Payables · VAT · Reports
- **Communication** — WhatsApp · Email · Calls · Meetings · Campaigns
- **Marketplace** — Lead Marketplace · Service Marketplace · Provider Directory
- **Industry Clouds** — Medical · Recruitment · Real Estate · Logistics
- **Administration** — Users · Roles · Audit Logs · API Keys · Security · Settings

## Grundregel (the one rule)

Every new feature, plugin, module and industry cloud MUST subordinate to one of
these lifecycle categories. Nothing gets its own top-level silo. This keeps the
platform legible and consistent even at 100+ modules.

## Dashboard logic (every dashboard answers four questions, in order)

1. **What happened?** → KPIs.
2. **What needs attention?** → alerts, overdue items, risks.
3. **What must I do today?** → meetings, tasks, calls, approvals (work queue).
4. **Which data do I manage?** → tables, lists, records (the launchpad).

## UAE-first

Xentral is UAE-first: bilingual EN/AR with full RTL, AED currency and UAE formats,
FTA/VAT-aware finance, and the four target verticals (Medical, Recruitment, Real
Estate, Logistics) as Industry Clouds under the lifecycle structure above.
