Go-live step 3 (seam): branded /auth/login page (Fiori + Inter, matches the live app) and a dormant SessionPort resolver. The resolver registers only when XENTRAL_LIVE_DATA=1 and XENTRAL_SESSION_SECRET are set; it then reads an HMAC-signed "xentral_session" cookie and returns { userId, companyId, role } as the tenant scope. No secret/cookie or a bad signature → null → seed. The public preview keeps both unset, so currentScope() stays undefined and every page renders seed data. This is the seam the existing Xentral login plugs into at cutover.
